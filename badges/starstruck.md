# Starstruck

## What It Is
Awarded when one of your repositories receives a significant number of stars. Has four tiers.

## Tiers

| Tier | Stars Required on a Single Repo |
|------|--------------------------------|
| Bronze | 16 |
| Silver | 128 |
| Gold | 512 |
| Platinum | 4096 |

> Stars must be on a **single repository** — they do not add up across repos.

## How to Earn It

You cannot directly "do" something to earn this — you need other GitHub users to star your repo.

**To maximize stars:**

### 1. Build something useful
- Developer tools, CLI utilities, templates, boilerplates
- Things that solve a real problem people search for

### 2. Write a great README
- Clear title and description
- Screenshot or demo GIF at the top
- Quick install/usage instructions
- Badges (build status, license, npm version)

### 3. Share it everywhere
- Post on [Reddit](https://reddit.com/r/programming) or [r/webdev](https://reddit.com/r/webdev)
- Share on LinkedIn with context about what problem it solves
- Tweet with relevant hashtags (#opensource #github #devtools)
- Submit to [Hacker News](https://news.ycombinator.com) (Show HN posts)
- Add to [awesome lists](https://github.com/sindresorhus/awesome) in your domain

### 4. Hacktoberfest
- Participate every October — active repos get visibility

### 5. Timing
- Post on Tuesday–Thursday, 9am–12pm your audience's timezone
- Avoid weekends and holidays

## What Helps Legitimately

- Open sourcing a useful side project
- Writing a tool that fills a gap no other library fills
- Contributing to trending topics (AI tools, dev productivity)
